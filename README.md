# ⚗️ GC Multi File Analyzer

A web app for analyzing gas chromatography traces, matching peaks to known compounds, and exporting publication-ready data tables.

---

## Quick Start

### macOS / Linux
```bash
bash run.sh
```
Then open **http://localhost:8501** in your browser.

### Windows
Double-click **run.bat** (or run it from a terminal).

### Manual start
```bash
pip install streamlit pandas numpy scipy openpyxl matplotlib
streamlit run app.py
```

## Workflow

### Step 1 — Compound Library
Prepare a CSV (or Excel) file with your known compounds. Download the template from the sidebar.

| Column | Required | Description |
|---|---|---|
| `compound_name` | ✅ | Unique name for the compound |
| `retention_time` | ✅ | Expected elution time (minutes) |
| `functional_group` | — | Group label for aggregated output (e.g. "Alcohol") |
| `window` | — | ± RT tolerance in minutes (default: 0.1) |
| `mw` | — | Molecular weight in g/mol (required for molar/mass yield) |
| `density` | — | Density in g/mL (optional, for reference) |
| `is_internal_standard` | — | TRUE/FALSE — marks the IS compound |
| `response_factor` | — | Relative response vs IS (default: 1.0) |

### Step 2 — Internal Standard
If you have an IS, enter the **moles of IS added** to each injection. This unlocks molar yield and mass yield output columns.

### Step 3 — GC Trace Files
Upload one CSV (or Excel) file per sample. Each file should have:
- Column 1: retention time (minutes)
- Column 2: detector signal (any units)

Headers are auto-detected. Files can be named anything — the filename becomes the sample name.

### Step 4 — Peak Detection Settings
Adjust these in the sidebar if peaks are being missed or over-detected:

| Parameter | Default | Effect |
|---|---|---|
| Min height % | 0.5% | Ignore peaks below this fraction of the tallest peak |
| Min prominence % | 0.2% | Required prominence relative to surrounding baseline |
| Min width (pts) | 3 | Minimum peak width in data points |
| Smoothing window | 11 pts | Savitzky-Golay window for noise reduction |
| Baseline window | 5% | Rolling minimum window for baseline subtraction |

### Step 5 — Output Options
- **Group by:** Compound (individual) and/or Functional Group (aggregated)
- **Value types:** Raw area, Normalized area (÷ IS ÷ RF), Molar yield (mmol), Mass yield (mg)

## Output Files

| File | Format | Best for |
|---|---|---|
| `gc_results.xlsx` | Multi-sheet Excel workbook | Excel figures, sharing |
| `gc_results_csvs.zip` | One CSV per table | Origin, Prism, Python |
| `gc_raw_peaks.csv` | Long-format, all peaks | Custom analysis |

All wide-format tables have **compounds/groups as rows** and **samples as columns** — ready to paste directly into Origin as a column plot dataset.

## Algorithm Details

1. **Baseline correction** — Rolling minimum subtraction (window = configurable % of trace length). Removes detector drift and solvent baseline without distorting peak shapes.

2. **Smoothing** — Savitzky-Golay filter (order 3). Reduces noise while preserving peak widths and heights better than simple moving average.

3. **Peak detection** — `scipy.signal.find_peaks` with height, prominence, and width thresholds. Prominence filtering is key for resolving partially overlapping peaks.

4. **Peak boundaries** — `scipy.signal.peak_widths` at 99.5% relative height to find valley-to-valley integration limits.

5. **Integration** — Trapezoidal rule on the baseline-corrected (not smoothed) signal within the detected bounds. Uses the raw signal for maximum accuracy.

6. **Compound matching** — Each detected peak is compared to the library retention times. Assignment is greedy: peaks are processed largest-area-first, and each compound can only be matched once per sample. Peaks outside all compound windows are silently ignored.

7. **Yield calculations:**
   - Normalized area = (compound area / IS area) / response_factor
   - Molar yield (mmol) = normalized_area × IS_moles × 1000
   - Mass yield (mg) = molar_yield × compound_MW
